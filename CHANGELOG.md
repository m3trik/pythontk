# pythontk — Changelog

## 2026

- **`ImgUtils.compute_atlas_layout` + `assemble_atlas` — texture-atlas layout & compositing primitives (for lightmap atlasing).** Two DCC-agnostic pieces that turn N weighted items into one packed atlas. **`compute_atlas_layout(weights, rows=None)`** is a deterministic shelf packer: it returns one `(scaleX, scaleY, offsetX, offsetY)` rect per item that **tiles the unit square exactly** (no gaps/overlaps), with each rect's *area* proportional to its weight so a large object gets more atlas texels — exactly the `scaleOffset` form an atlas needs and Unity's `Renderer.lightmapScaleOffset` convention (`uv' = uv*scale + offset`). Items are balanced across `round(√n)` shelves by longest-processing-time to keep aspect ratios sane; all-zero weights fall back to equal shares, negatives clamp to 0. Pure Python (no numpy/PIL). **`assemble_atlas(images, rects, size, background=0)`** composites each item's image into its rect on one canvas, resizing (cv2; `INTER_AREA` downscale) and applying the standard UV→row **vertical flip** (UV v=0 bottom ↔ image row 0 top) so an item later bound with the same `scaleOffset` samples the pixels written; HDR-safe (float32), channel/dtype preserved, degenerate (zero-area) rects skipped. Consumed by mayatk's `LightmapBaker.pack_atlas` (Atlas-by-Material packing). Tests: `test/test_img.py::AtlasLayoutTest` (exact tiling, area∝weight, order preserved, zero-weight-shelf guard, n up to 37) + `AtlasAssembleTest` (UV-flip placement, dtype/size, zero-rect skip, grayscale 2D round-trip) — +15, full img suite 121 green.
- **Per-map output-format templates — save each map in its target profile's exact container + bit depth.** New `OutputSpec(ext, bit_depth, compression)` + `OutputTemplate` (`img_utils/output_template.py`), with the built-in per-`WF`-profile catalogue and resolution encapsulated in an `OutputTemplates` registry class (`OutputTemplates.resolve(map_type, profile)` / `.get(profile)` / `.BUILTIN` / `.DEFAULT`); plain-data (`to_dict`/`from_dict`) so a future editable `PresetStore` tier can wrap them as the built-in tier. `ImgUtils.save_image` gained **`bit_depth=`** (16-bit PNG/TIFF — grayscale via PIL `I;16`, RGB via cv2 `uint16`; 8-bit sources promoted ×257; containers that can't hold 16-bit degrade to 8-bit + warning) and **`compression=`** for DDS block formats (`DXT1/DXT3/DXT5/BC5` via Pillow now; `BC7/BC6H` route to an optional codec registered through `ImgUtils.register_dds_codec`, raising a clear error until an extension installs one). The pipeline applies a profile's template **only when `output_profile` is set** — `TextureProcessor.save_map` (the per-map save chokepoint), `MapOptimizer.optimize_map`, and `MapCompositor` each resolve per-map ext/bit-depth/compression from it; behavior is **unchanged when unset** (fully backward-compatible). This is the engine/SSoT layer (built-in tier); editable templates + panel profile selectors are the follow-up. Tests: `test/test_output_template.py` (resolution: per-profile, override/default fallback, unknown profile/map, dict round-trip) + `test/test_img.py` (16-bit grayscale/RGB round-trips, unsupported-container fallback, DXT5, BC7 clear-error + codec hook).
- **`ImgUtils` image-format capability table + DDS support — one SSoT for IO routing.** Replaced the flat `texture_file_types` list with `image_formats` (`{ext: ImageFormat(read, write, backend)}`) and *derived* `recognized` / `readable` / `writable` from it so the sets can't drift; `texture_file_types` remains as a back-compat alias of `recognized`. `save_image` / `load_image` now dispatch on each format's `backend` — Pillow for most, OpenCV for the float formats (EXR, HDR) Pillow can't encode/decode — via `_save_via_cv2` / `_load_via_cv2`; a read-only or unknown format raises a clear `ValueError` instead of a cryptic `KeyError`. **DDS** is now a recognized read+write format (Pillow's DXT-tier writer; BC7/BC6H still raise loudly). Every previously divergent writer was unified through `save_image` — `MapOptimizer.optimize_map`, the seven direct `Image.save` calls in `MapFactory`, and `ImgUtils.pack_channels` / `extract_channels` — so EXR/HDR/DDS output works everywhere and there is a single save path. `OPENCV_IO_ENABLE_OPENEXR` is set at module import (OpenCV caches it at first codec init). Consumed by mayatk (`GameShader`, `MaterialUpdater`) and extapps (`map_converter`), which now build their convert-format menus from `writable`. Tests: `test/test_img.py::ImageFormatCapabilityTest` (table/derived-set integrity, alias preserved, DDS+EXR+HDR round-trips, read-only-format guard, `get_images` survives an EXR in the scanned dir).
- **`ImgUtils.dilate_image` — texture edge-padding / gutter fill (pure numpy, HDR-safe).** `dilate_image(image, mask=None, iterations=-1, connectivity=)` extends valid pixels outward into empty (background) regions so bilinear filtering and mip generation never pull background color across a UV-island seam — the standard "dilation" / "edge padding" baking tools apply. Each pass assigns every empty pixel adjacent to filled pixels the average of its filled neighbors (`iterations=-1` fills entirely; a bounded count gives a fixed gutter width); numpy-in/numpy-out, same shape/dtype, no PIL/cv2 dependency so it works on float HDR data. The optional `mask` is the key contract: it defaults to "any channel > 0", but baked-map callers must pass the explicit **coverage** mask — a luminance heuristic wrongly treats dark-but-valid texels (shadow contact, near-black albedo) as empty and overwrites them. Consumed by mayatk's `LightmapBaker` (mask = Arnold RTT alpha). Tests: `test/test_img.py::DilateImageTest` (single-source fill; dark-valid-pixel preserved as a fill source under an explicit mask vs. overwritten under the default; iteration bound; shape/dtype/2D; all-empty terminates; mask-shape mismatch raises).
- **`MathUtils.catenary_sag` — added a `gather` push-pull lobe at the supports.** The existing `round_amount` only *eases* the cusp into a smooth dome (it reduces sag near each support but never rises above the baseline or dips back). The new `gather` arg (default `0`, orthogonal to `round_amount`) adds the missing overshoot: the `ring²·(2.4 − 3.4·ring)` lobe (`ring = cos²(πs)`) lifts the profile **above** the baseline right at the support (−1 sag-unit at `gather=1` — a gathered-header pucker rising above the rail) and **dips** just inside it (`+0.18`) as the slack falls off, easing back to `0` by mid-span so the center peak is untouched. Consumed by mayatk's Curtain generator as its new push-pull *Gather* dial. Tests: `test/test_math.py` (push-above-baseline at the support, pull-extra-sag inside, center-untouched, and `gather=0` matches the plain catenary).
- **`MathUtils.simplify_rdp` + `point_segment_distance` — curve simplification / adaptive sampling.** **`simplify_rdp(points, tolerance)`** is the Ramer-Douglas-Peucker algorithm: it returns the indices of the polyline vertices to *keep* so the simplified polyline stays within `tolerance` of the original, concentrating the kept points where the polyline bends (corners, tight arcs) and collapsing near-collinear runs to their endpoints — the standard, error-bounded way to decide *where to spend* samples/divisions along a curve. Iterative (explicit stack, safe on dense polylines), any fixed dimension, monotone under tolerance, endpoints always retained; `tolerance <= 0` or `< 3` points returns everything. Backed by **`point_segment_distance(p, a, b)`** (perpendicular distance to a segment, endpoint-clamped, n-dimensional) — a reusable primitive alongside the existing point/polyline helpers. Consumed by mayatk's Curve to Tube to place polygon rings on the bends of the path. Tests: `test/test_math.py` (segment distance perpendicular/clamped/degenerate; RDP collapses straights, keeps corners, concentrates on bends, 2D, edge cases).
- **Math/signal primitives promoted from the mayatk Curtain generator.** Reusable, DCC-agnostic pieces that were private to `mayatk.edit_utils.curtain` now live at the SSoT layer. `MathUtils` gains: **`smoothstep(x, edge0=0, edge1=1)`** (the canonical *clamped* Hermite — distinct from `ProgressionCurves.smooth_step`, which applies a weight exponent and doesn't clamp), **`ricker(x)`** (Mexican-hat wavelet — a zero-integral ridge-with-troughs, for mean-preserving bumps/creases/folds), **`catenary(t, tension)`** + **`catenary_sag(t, tension, round_amount=0)`** (normalized `cosh` sag profile for cables/ropes/cloth, optionally rounded toward `sin²` at the supports), **`safe_normalize(vector, fallback)`** (zero-guarded `normalize`), and **`lerp`** now interpolates point/vector sequences component-wise in addition to scalars. New **`BandLimitedNoise`** (`math_utils/noise.py`, exported at top level): coherent multi-octave sine noise (1/f roll-off, deterministic from `seed`, optional `u_periodic` seam-wrap) — the opposite of per-vertex white noise, for procedural surface relief (cloth grain, gentle terrain, displacement). Tests: `test/test_math.py` (the new MathUtils helpers) and `test/test_noise.py` (determinism, coherence, amplitude bound, periodic wrap).
- **`PresetStore` — Qt-free, two-tier named-preset store (built-in + user).** Where `UserConfig` resolves a *single* config doc, `PresetStore` manages a *collection* of named JSON presets across a read-only **built-in** tier (shipped in a repo `presets/` dir next to a module) and a writable **user** tier under `user_config_root()` (the same consolidated root uitk's `PresetManager` uses). A user preset **shadows** the built-in of the same name (replace, not merge — "duplicate to edit" covers tweaking a shipped default), so `list()` shows each name once and `load()` returns the user copy when present. API: `list(tier=None|"user"|"builtin")`, `source(name) → "user"|"builtin"|None`, `exists`, `load` (KeyError on missing), and `save`/`delete`/`rename` operating on the user tier only (built-ins stay read-only); `sanitize_preset_name` is the shared name→filename rule. It deals only in plain dicts and imports no Qt, so headless engines (e.g. the photogrammetry runners in Metashape's bundled Python 3.9) use it directly — and uitk's `PresetManager` now delegates its storage/discovery to it, so the GUI and headless paths share one set of dirs, names, and rules. Exposed at the top level (`pythontk.PresetStore`, `pythontk.sanitize_preset_name`). Tests: `test/test_preset_store.py` (tier union + user-shadowing, source/exists, load-prefers-user + KeyError-on-missing + available-list, save/delete/rename user-only with built-in read-only protection, shared name sanitization, default user dir under `user_config_root`).
- **Marmoset Toolbag engine removed from `pythontk` (`marmoset_utils` deleted).** `pythontk` is the zero-dependency, DCC-agnostic core; a Toolbag-SDK integration engine is domain glue, not a generic primitive — it was an over-centralization (see the earlier "Marmoset Toolbag engine" entry below, now superseded). `MarmosetEngine` is no longer exported at the top level. The engine + `templates/` + `_toolbag_helpers` + `toolbag_log` + `template_params` + the `marmoset_rpc` live-session client now live with each consumer: `mayatk.mat_utils.marmoset_bridge` (full, incl. `marmoset_rpc`) and the standalone `extapps.marmoset_workflow` panel (engine subset — `import`/`lookdev`, no bake, no RPC). The genuinely-generic primitives it builds on stay upstream and unchanged: `pythontk.net_utils.rpc.RpcClient` (JSON-RPC base) and `uitk.bridge` (param widgets + `__KEY__` template rendering). Mirrors how `substance_bridge` / `substance_workflow` are already handled. The mock tests moved to `mayatk/test/mock_tests/` (`test_toolbag_helpers.py`, `test_marmoset_rpc.py`).
- **`ImgUtils.get_image_size(path)` — zero-dependency image dimensions.** Reads `(width, height)` straight from the JPEG/PNG header with the **stdlib only** (a `_image_size_from_header` parser: PNG IHDR + JPEG SOF scan, guarded against a malformed-segment loop), falling back to PIL for other formats when available. Complements `get_image_info` (PIL-based) for the case where you need only the dimensions and **can't assume PIL/cv2/numpy** — notably Metashape's bundled Python, where `extapps`' `--texture-size auto` was silently falling back to its 8192 default instead of the source's true resolution. Test: `test/test_img.py::ImgTest::test_get_image_size_header_parse` (non-square PNG+JPEG; garbage → `None`).
- **`ExposureEqualizer` — near-lossless, EXIF-preserving output (was silently degrading the set).** The equalizer re-saved every frame via `cv2.imwrite` at its default JPEG quality (~95) with chroma subsampling and **no EXIF** — so an equalized set fed to alignment carried generational re-compression artifacts (which hurt SfM feature matching) and had lost its focal-length / camera priors. `equalize_directories` now takes `quality` (default `100`) and `preserve_exif` (default `True`); JPEG output is a single high-quality 4:4:4 encode that carries the source EXIF through (via PIL when available, else a `cv2` high-quality fallback), and PNG/TIFF stay lossless. Backward-compatible (new kwargs default to the safer behavior). Frames are read with `IMREAD_IGNORE_ORIENTATION` so the saved pixels stay in the same frame as the copied `Orientation` tag — otherwise cv2's default auto-righting + the verbatim tag would double-rotate the image in any EXIF-aware consumer (RealityScan / Metashape). Docstring now notes equalization's payoff is seam-free texturing across **multiple** captures — for *alignment*, prefer leaving radiometry to the SfM tool. Test: `test/test_vid.py::ExposureEqualizerTest::test_equalize_preserves_exif`.
- **`ImageCurator` — median-relative blur guard (`min_sharpness_fraction_of_median`).** The percentile floor only removes a fixed bottom slice of the survivor set, so when a capture has more blur than that slice, boundary-case *catastrophically* defocused frames slip through (found in testing: a frame at ~0.3% of the set median survived). The new guard (`curate`/`preview`, 0–1, default `0.0` = off) additionally drops any cluster representative below that fraction of the **survivor-median** sharpness — self-calibrating, near-dup blur can't dilute it, and the strictest of the three floors wins. Off by default so existing consumers are unchanged; the photogrammetry workflows enable it at `0.15`. New test in `test/test_vid.py` (`test_median_fraction_guard_drops_catastrophic_blur`).
- **`UserConfig` — Qt-free, zero-dependency user-config resolution (`core_utils/user_config.py`).** Lets a package ship a generic default + an `*.example.json` template while the user's personal/site values live in a JSON file **outside** the package source. `UserConfig.resolve(name, package=, env=, default=, path=)` discovers the source file (explicit `path` → `$env` → `<user_config_root>/<package>/<name>.json` → none) and **deep-merges** the user doc over the default, so a partial profile need only carry overrides; `expand()` resolves `~` / `${ENV}` in path values. `user_config_root()` reproduces uitk's consolidated config location (`%LOCALAPPDATA%/uitk`, `~/.config/uitk`, `~/Library/Preferences/uitk`; override `UITK_PRESETS_ROOT`) **without importing Qt** — uitk's `SettingsManager`/`PresetManager` are `qtpy`-bound and unusable from headless contexts (e.g. the photogrammetry engines inside Metashape's bundled Python 3.9), so this honors the same root *by convention* and the two stay in lockstep without a dependency. JSON, not TOML (`tomllib` is 3.11+; won't import under 3.9). Used by `extapps.photogrammetry_profile`. Tests: `test/test_user_config.py`.
- **`AppLauncher` — interactive-session launch + detection (for headless DCC/SDK driving).** New `current_session_id()` / `active_console_session_id()` / `is_interactive_session()` (Windows session awareness — session 0 is the non-interactive services session with no window station) and `launch_in_session(app, args, session=None, ...)`, which launches a window-station-bound app into the **active console session** from a non-interactive context (SSH / service session 0) via Sysinternals PsExec. PsExec is *discovered* (`find_session_launcher`: explicit → `$PSEXEC` → PATH → tool dirs), never bundled — pythontk stays zero-dependency. `run()` also gains additive `output_file` (stream stdout+stderr to a file instead of buffering huge logs in memory) and `env`. All additions are backward-compatible. Used by the new `extapps` Metashape/RealityScan launch connections. Tests in `test/test_app_launcher.py` (`TestAppLauncherSessions`).
- **`ImageCurator.preview()` — dry-run curation report.** Scans a set **once** and evaluates one or more dHash `hash_thresholds` *without copying any files*, returning `{n_scanned, sharpness:{min,p5,p25,median,p75,max}, thresholds:[{hash_threshold, n_clusters, n_kept, reduction_pct, sharpness_floor}]}` — so curation can be tuned on a real capture before committing a long SfM run. The scan / cluster / select stages were extracted into shared `_scan_images` / `_cluster` / `_select` helpers (DRY — `curate()` and `preview()` now share one implementation; no behavior change to `curate()`). Test: `ImageCuratorTest.test_preview_reports_survivors_without_copying`.
- **Photogrammetry prep — sharper, portable culling + gentler equalization.** (1) `ImageCurator.curate()` gains `sharpness_floor_percentile` (0–100): derives the blur cutoff from the variance-of-Laplacian distribution of the **cluster representatives** (the sharpest-per-cluster frames actually headed downstream) instead of a scene-dependent absolute constant, so the same setting drops "the blurriest ~10%" of the survivor set on any capture. Computing it over representatives rather than every scanned frame is deliberate — a large mass of blurry near-duplicates (e.g. a handheld floor-pan segment) would otherwise drag an all-frames cutoff down and under-cull. Applied after per-cluster top-K selection, so only uniformly-blurry clusters are culled; parallax-bearing sharp views survive. (2) `ExposureEqualizer.equalize_directories()` gains `strength` (0–1, blends the Reinhard match with each frame's original in LAB so local contrast isn't flattened — right before tools that re-balance internally, e.g. RealityCapture MosaicBlending) and `reference_strategy` (`first`/`median`/`global`) so the target distribution isn't forced to the first capture's color cast (`median` picks the dir nearest the set-wide mean luma; `global` averages all). Defaults preserve prior behavior (`strength=1.0`, `reference_strategy="first"`). New tests in `test/test_vid.py` (`ImageCuratorTest`, `ExposureEqualizerTest`).
- **Marmoset Toolbag engine** (`marmoset_utils`) — the DCC-agnostic Toolbag integration carved out of `mayatk.mat_utils.marmoset_bridge`. `MarmosetEngine` launches Toolbag and runs a rendered template (`send_to` interactive / `roundtrip` headless) against a plain **model path** + optional materials-manifest + plain params dict, so any host composes it (the Maya bridge, the new `extapps.marmoset_workflow` panel, a CLI). Ships `toolbag_log` (log resolution/classification/tail), `template_params` (UI-free default token values), the in-Toolbag `_toolbag_helpers`, the `templates/` scripts, and the `marmoset_rpc` live-session client (a `RpcClient` binding). `render_template`'s first path kwarg is now `model_path` (was `fbx_path`); the template token is `__MODEL_PATH__`. `pythontk.MarmosetEngine` is exposed at the top level. Moved-code tests live in `pythontk/test/` (`test_toolbag_helpers.py`, `test_marmoset_rpc.py`; 96 pass).
- **Python 3.9 compatibility restored** (matches the declared `requires-python = ">=3.9"`). Four modules used PEP 604 unions (`X | Y`) in annotations without `from __future__ import annotations`, which raises `TypeError` at import time on 3.9 — breaking hosts that ship a 3.9 interpreter (notably Agisoft Metashape, whose bundled Python runs the `extapps` photogrammetry engines). Added the future import to `net_utils/credentials.py`, `core_utils/logging_mixin.py`, `core_utils/execution_monitor/_dialog_viewer.py`, and `core_utils/execution_monitor/_execution_monitor.py`. New `test/test_py39_compat.py` statically enforces the contract (PEP 604 annotations require the future import; no runtime unions; no `match` statements) so it can't silently regress. Verified by importing all 78 modules under Metashape's bundled Python 3.9.13.
- **Structural consolidation (no public-API change).** Reduced localized bloat without touching the public surface (`from pythontk import ...` and module paths for `MapFactory`/`TextureProcessor`/`ConversionRegistry`/`HierarchyDiff` resolve as before; full suite green):
  - `img_utils/map_factory.py` (4,428 lines) split into a `map_factory/` package — `conversions` (registry) → `processor` (`TextureProcessor`) → `handlers` (workflow strategies) → `_map_factory` (orchestrator). `MapFactory` is late-bound into `processor`/`handlers` by the package `__init__` to break the runtime cycle their primitive calls would otherwise create.
  - `core_utils/hierarchy_diff.py` moved into the `hierarchy_utils/` subpackage so all hierarchy code lives together. `HierarchyDiff` (aggregate result container) and `HierarchyAnalyzer`/`HierarchyDifference` (engine + per-record) are complementary, not duplicates, so they were **not** merged.
  - Fixed dead `DEFAULT_INCLUDE` entry `core_utils.table_mixin` → `TableMixin` (the class lives in `logging_mixin`); `pythontk.TableMixin` now resolves instead of returning nothing.
  - Removed dead code: orphan `file_utils/mesh_convert/mesh_convert_ui.py` (unreferenced PySide6 import — violated the zero-Qt rule) and the unused `core_utils/hierarchy_utils/backup/` snapshot.
- **`TextureOptimizer` → `MapOptimizer`** (breaking) — renamed `img_utils/texture_optimizer.py` → `img_utils/map_optimizer.py`, class `TextureOptimizer` → `MapOptimizer`, methods `optimize_texture` → `optimize_map`, `batch_optimize_textures` → `batch_optimize_maps`. Brings the module into line with its `map_*` siblings (`map_compositor`, `map_factory`, `map_registry`). No back-compat shim — downstream callers (`mayatk`, `extapps`, `uitk` docstrings) updated in lockstep.
- **Removed `ImgUtils.map_modes` / `ImgUtils.map_types` / `ImgUtils.map_backgrounds`** (breaking) — these registry-derived class attrs invited a backwards `_img_utils → map_registry` module-load coupling. `MapRegistry` is the sole source of truth; access via `MapRegistry().get_map_modes()` etc. (singleton, cheap). `ImgUtils.set_bit_depth` and `ImgUtils.get_base_texture_name` use a deferred local import so generic `ImgUtils` consumers no longer pay the map-registry instantiation cost on import.
- **MapCompositor engine** (`img_utils.map_compositor`) — multi-layer texture compositor with auto DirectX/OpenGL normal-map complement generation. Moved here from the standalone `map_compositor` repo; the UI panel now ships in `extapps`.
- **FrameExtractor** (`vid_utils.frame_extractor`) — OpenCV-backed video → still-frame extraction. Carved out of the deprecated `metashape_workflow` repo as a generic helper.
- **UI panels removed** (clean cut, no shims): `map_converter`, `map_packer`, `mesh_convert` Switchboard slot classes / `.ui` files have moved to `extapps`. Engine logic (`ImgUtils`, `MapFactory`, `MeshConvert`) stays here. `pythontk.MeshConvertSlots` and `pythontk.MeshConvertUi` no longer exist at the top level — import the panel from `extapps.mesh_convert` if needed.
- **Pillow** added as a declared dependency (already used transitively by `ImgUtils`, `MapFactory`, `MeshConvert`, and `_gif_viewer`; declaration was overdue).
- **Affix hardening** — `StrUtils.strip_known_affix` / `StrUtils.apply_affix` idempotently strip and reapply a configured prefix/suffix (case-insensitive, separator-tolerant including stray leading/trailing `_`, no false-positives like `Matte_door` for prefix `Mat_`). `strip_known_affix` is a conservative primitive — it touches only the affix and adjacent separators, never internal/remote underscores. `apply_affix` is a no-op when both affixes are empty (so user-typed names pass through untouched), and trims dangling underscores only on the side(s) where an affix is being applied. `ImgUtils.get_base_texture_name`, `MapFactory.get_base_texture_name`, `MapFactory.group_textures_by_set`, `MapFactory.resolve_texture_filename`, and `MapFactory.prepare_maps` now accept `prefix`/`suffix` and route them through, so callers can safely round-trip texture/material names without producing `Mat_Mat_<name>` duplicates or trailing-underscore artifacts.
- **AudioUtils module** — reusable ffmpeg-backed audio conversion helpers (`ensure_playable_path`, extension checks, ffmpeg resolution) for downstream DCC tools.
- **Audio Composite Builder** — DCC-agnostic WAV compositing helper (`build_composite_wav`) — centralizes timeline-audio mixing logic.
